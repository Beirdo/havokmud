Wizhelp Edit

The edit command is very useful and extremely powerful once you become familliar with it.  It takes a little while at first, but once you've got a bit of building experience behind you, the commands will fly from your fingers, and your rooms will get built that much faster.  This file is essentially a prettied-up version of wizhelp edit that I did a few years ago as a standalone file.  However, now that I have an actual Builders' Manual online, I figure this would be a good place in include it.

If you forget a specific command with edit, or a sequence of variables, or whatnot, look this up and you can probably figure it out pretty quick.

 

The context of 'edit' is as follows:

Edit [field] [attributes]

The context of 'edit field' is as follows:

  • Name - This defines the name of the room.
  • Desc - This brings up a prompt which allows you to change the description of a room.
  • Exit - This adds, deletes, or modifies the exits of the room.
  • FS - This defines the Flags and Sector type of the room.
  • Exdesc - This adds, changes, or deletes extra descriptions in the room.
  • Extra - This adds an extra description in the room.
  • Riv - This defines perameters used for flowing rivers.
  • Tele - This defines perameters used for teleporting rooms.
  • Tunn - This defines perameters used for tunnel rooms.

    The context of edit attributes is as follows:

  • Name - [name of room]
  • Desc - No attributes, here. It will prompt you for text.
  • Exit - [Direction][Door Flags][Key#][To-Room#][Open Commands]
  • FS - [Flags][Sector Type]
  • Exdsc - [Direction] (It will prompt you for text.)
  • Extra - [Keywords] (It will prompt you for text.)
  • Riv - [Speed][Direction]
  • Tele - [Time][To-room#][Look-Flag]
  • Tunn - [Mob-Limit]

    Definitions of Context:

  • [Dir]
    • 0 = North
    • 1 = East
    • 2 = South
    • 3 = West
    • 4 = Up
    • 5 = Down
  • [Door Flags]
    • 'S' = Secret (Makes the exit not appear in the list of exits)
    • 'C' = Climb (Makes the player climb or fly to pass through the exit)
    • 'P' = Pickproof (and bashproof, knockproof) (Makes the player have the key to unlock the door)
    • 'D' = Is-Door (Puts a door in the exit. Otherwise, it's just an open exit)
    • **Note** Not all possible Door Flags are listed here. To use others, such as 'Closed' or 'Locked' you must use 'Redit.'
  • [Key#] The Key# is the vnum of the object that will unlock the door. If there is no door, use -1. If the door cannot lock, use 0.
  • [To-Room] The Room# of the room that the exit leads to.
  • [Open Command] The command the player must type in to open or close the exit.
    • Pull = 224
    • Twist = 371
    • Turn = 372
    • Lift = 373
    • Push = 374
    • Use the listed number for [Open Command]
  • [Flags]
    • Dark = 1
    • Death = 2
    • No_Mob = 4
    • Indoors = 8
    • Peaceful = 16
    • Nosteal = 32
    • No_Sum = 64
    • No_Magic = 128
    • Tunnel = 256
    • Private = 512
    • I haven't tested this yet, but I imagine that to use multiple Flags you add the numbers together, since that's how everything else works.
  • [Sector Type]
    • Indoors = 0
    • City = 1
    • Fields = 2
    • Forest = 3
    • Hills = 4
    • Mountains = 5
    • Water (swimmable) = 6
    • River (non-swimmable) = 7
    • Air = 8
    • Water = 9
    • Desert = 10
  • [Time] The time interval between Teleports. This must be an interval of 10. 10 is very fast, 60 is very slow.
  • [Look Flag] If this number is 0, the player will not know when they are teleported. If it is not 0, they will see the room when they are teleported to it.
  • [Keywords] The word that the player must type to see the info. Ex: if the keyword is 'case' then the player must type 'exa case.'
  • [Speed] Must be a mulitple of 15. Lower is faster. If you don't want it to flow, use 0.
  • [Mob Limit] This sets the number of mobs that can be in a room at once. The Tunnel Room Flag must be set on. 'Edit Tunn X' will do this automatically.
