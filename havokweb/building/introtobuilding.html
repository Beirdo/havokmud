Intro to Building

  For a complete walk-through, check out the Guide to Building on Havok: these pages are intended as quick-references.

 I.1: Intro to the Building Commands

Here is a quick list of all of your Level 53 commands.  Keep in mind, you can type wizhelp (command-name) to get in-game help on a given command.  These will all be gone over in further detail later on.

 The general rule is: if a command starts with O, it’s for objects, R for rooms, and M for mobs.

Commands for editing rooms: 

Redit  is the easiest way to edit a room.  Use this when you’re beginning.   It will open a VT100 term.

Rload will reset a room to the last saved state.  Use this if you mess up.

Create is used to create new rooms.

Edit is the best way to edit most features of a room.  It is command line-based, and is an advanced tool, but powerful and fast.

Rsave is used to save a room.  REMEMBER TO SAVE!

 

Commands for editing objects:

 

Oedit is the best way to edit an object.  It will open a VT100 term.

Ooedit is the worst way to edit an object.  It is command-line oriented, and I won’t go into it in this document.

Setobjmax is used to set a ‘max’, which limits the number of the item that can exist in the game at once.  DON’T USE THIS WITHOUT PERMISSION.

Osave is used to save an object.  REMEMBER TO SAVE!

Setobjspeed is used to set a weapon’s attack speed.

 

Commands for editing mobiles:

 

Mforce will force a mob to do what you tell it.

Msave is used to save a mob.  REMEMBER TO SAVE!

Medit  is used to edit a mob.

 

Misc. commands that are useful when building:

 

String  is an advanced command to modify objects and mobs.

Show will show lists of mobs, rooms, or zones.  Usage:

                       Show m (zone# or keyword) ex: show m 126, show m elf

                       Show o (zone# or keyword) ex: show o 126, show o bread

                       Show r (zone#)

                       Show z (to get a list of all the zones in the game)

Stat  will give detailed information on any mob, object or room.  Usage:

                        Stat (object/mobname)

                        Stat room

Zload will reload the zone to its last saved state.

Zsave will save mob and object locations.  REMEMBER TO SAVE!

Load is used to load a fresh mob or object.  Usage:

                        Load o (objectname or vnum)

                        Load m (mobname or vnum)

Purge will destroy any mob or object in your inventory or in the room.

Zclean  will destroy all mobs and objects in the zone you’re standing in.

 

I.2: Intro to Building Terminology

 

Vnum:  ‘vnum’ stands for virtual number.  A zone’s boundaries are defined by what vnums are assigned to it.  For instance, a zone might be assigned vnums 1001 through 1100.  More likely, you’ll get a zone with vnums in the tens of thousands.  Havok is big.  Each vnum can store one room, one object, and one mob.  That is to say, if you were assigned 1001-1100, you can save one hundred rooms, one hundred objects, and one hundred mobs within your zone, and while no two rooms will have the same vnum, nor objects, nor mobs, there can be one room, object, and mob each with the vnum 1001.
