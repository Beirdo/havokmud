Exits and Doorways

  For a complete walk-through, check out the Guide to Building on Havok: these pages are intended as quick-references.

 

Exits allow players to move between one room and another.  There are many different types of exits, so you can set up each one exactly as you like it.  There are three ways to create an exit.  The first is with redit, using option 5.  This method is simple and straightforward, however it limits what you can do.  The best is with the edit command.  The third is the fast exits feature which, while fast and very handy, is very limited in what it can do.  We’ll go through all three.

IV.1: Making Exits With Redit

            To make an exit with redit, type redit while standing in the room as you did in Chapter III.  Enter 5 to open the exits menu.  There are three important pieces of information you need to know:

            Direction is the direction that the exit leaves the room from.

            To_Room  is the vnum of the room that the exit will lead to.

            Key#  is the vnum of the object that unlocks and locks the door, if  it has one.

IV.2: Making Exits With Edit

            The edit command is driven in a command-line manner.  You can type wizhelp edit for complete info, but I’ll reprint the important parts here.  Edit is a powerful command that can be used to edit pretty much anything in a room that you could want to.  We’ll go over the rest in Chapter V.  Here is how you make exits with it:

           

edit exit (dir) (door_flags) (key_number) [(open_command)]

 

            Now that probably doesn’t make a lot of sense immediately, so let’s go through it.  There are four different variables:

            Dir is the direction that the exit leaves the room from.

                                    0 is north.

                                    1 is east.

                                    2 is south.

                                    3 is west.

                                    4 is up.

                                    5 is down.

                                    To choose the direction, simply swap (dir) for 0-5 in the command above.

 

            Door_flags define what the exit IS, exactly.

                                    S is secret.

                                    C is climb (or fly).

                                    P is pickproof (and bashproof, knockproof).

                                    D is is_door, which means the exit has a physical door.

                                    X is null, meaning you don’t want any flags.

                                    Additional Notes: not every option is available here.  Some of them are available in redit.  To make a door closed, simply close it by hand.  To make a door locked, lock it by hand. 

 You have tohave a flag of some sort; use X if you want nothing.  The flags are cumulative; if you want more than one, stack them together, such as edit exit (dir) scp ….

           

            Key_number is the vnum of the object you want to be the key.  If you have no key, use 0.

 

            Open_commands are optional.  This is for if you want the door to be opened with a command other than open.

                                    Pull is 224.

                                    Twist is 371.

                                    Turn is 372.

                                    Lift is 373.

                                    Push is 374.

                                    To define a special open command, simply type in the appropriate number at the end of the edit exit string.

IV.3: Deleting Exits

            To delete an exit, use edit exit (dir) with no other modifiers.  It will delete the exit in that direction, and probably give you a screenfull of spam.  Don’t worry about it, the spam means it’s working!  ;^)

IV.4: Fast Exits

            To enable fast exits, type wizset fast.  This is a nice little feature that Banon has coded in for all of us.  While you have fast exits enabled, if you walk in a direction that has no exit, it will create an exit to the next available room that has not been created yet, and it will create an exit back.  It’s all very fast and automatic, but if you want anything other than a basic two-way exit, you’ll need to go back and modify them.

 

IV.5: Exit Descriptions

            Exit descs are special descriptions that are shown when a player attempts to look through an exit.  These are a very good way to add a bit of flavor to your zone, or to hide keywords to secrets, etc etc.  To make them, type edit exdsc (direction), where (direction) is 0-5 as listed in IV.2.

 

IV.6: Saving Exits

            There are two different things you have to do to save your exits, depending on how complex your exit is.  The first is rsave.  When you rsave, Havok will save all of the basic data that defines the exit.  However, if your door is closed or locked, then rsave will not remember this.  To save doors like this, you must zsave (zone#).  The basic reason for this is that Havok only loads room data once – when the game boots.  But things like whether or not a door is closed or locked are things that must be reset every time the zone resets – and when the zone resets, it reloads the zone data.  Therefore, zsave to save your exits being closed/locked.

            Note: Object and mob locations in your zone are also saved with zsave, so you should be careful when using it.  I personally recommend not having any mobs or items in your zone until the rooms are all built and set.  But many people prefer not to build this way.  If you have mobs and objects that are currently loading in your zone, type zclean (zone#) first – this will destroy all mobs and items in the zone, thus saving you from having all sorts of random things laying around (and trust me, THEY DO!).  Once the zone is clean, set all your doors and locks how you want them, then type zload (zone#) (which will load all your mobs and objects back), and zsave (zone#), which will resave everything, including your door info.
